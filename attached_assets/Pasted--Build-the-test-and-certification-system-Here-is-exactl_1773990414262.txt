> **Backend:**
> 1. Create a GET route `/api/questions/:courseId` — returns all questions for a course
> 2. Create a POST route `/api/test/submit` — accepts `userId`, `courseId`, and `answers` (array of selected answers). Calculate the score, determine if passed (3 out of 5 or more is a pass), save the result to `test_results`, and if passed create a certificate record in `certificates`. Return the score and pass/fail result.
> 3. Create a GET route `/api/certificates/:userId` — returns all certificates for a user
>
> **Frontend:**
> 1. On the `/learn/:courseId` page, once ALL lessons are marked as complete, show a "Take Final Test" button below the lesson list. This button should NOT appear until all lessons are completed.
> 2. Clicking "Take Final Test" takes the user to `/test/:courseId`
> 3. On the `/test/:courseId` page:
>    - Show the course name and instructions: "Answer all 5 questions. You need 3 or more correct to pass."
>    - Display all 5 questions one by one or all at once with 4 radio button options each
>    - A "Submit Test" button at the bottom
>    - On submit, call `/api/test/submit` with their answers
>    - If passed: show a success message "Congratulations! You passed. Your certificate is ready." with a "View Certificate" button
>    - If failed: show "You didn't pass this time. You scored X/5. Please review the lessons and try again." with a "Retake Test" button
> 4. The certificate page `/certificate/:courseId` should display a styled certificate showing:
>    - Techiehive logo/name in yellow
>    - "Certificate of Completion"
>    - Student's full name
>    - Course name
>    - Date issued
>    - A "Download Certificate" button that saves it as a PDF
